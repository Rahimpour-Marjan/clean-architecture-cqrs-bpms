# Enterprise BPMS Workflow Engine – .NET Sample
Real-world Clean Architecture + DDD + CQRS implementation  
(based on the actual Kardino BPMS platform used in production since 2020)

## Features
- Full Clean Architecture + Vertical Slice
- Domain-Driven Design (Aggregates, Value Objects, Domain Events)
- CQRS + MediatR pipeline with validation & logging behaviors
- Entity Framework Core + SQL Server (migrations, optimized queries)
- Background processing with Hosted Services & Hangfire
- Dynamic workflow engine: task routing, multi-level approvals, SLA tracking, document handling
- Highly modular & testable design

## Current Tech Stack
- .NET 6 LTS (or .NET 7 – whichever is correct)
- Entity Framework Core
- MediatR
- FluentValidation
- Hangfire
- SQL Server

> This is a sanitized, production-derived sample of the Kardino BPMS backend architecture (2020–2025).  
> Sensitive business logic and data have been removed or anonymized.
